SQL

mysql5.0以下和5.0以上的版本区别

5.0以下没有information_schema这个系统表，无法列表名等，只能暴力跑表名。
5.0以下是多用户单操作，5.0以上是多用户多操作

MySQL注入写shell

写入webshell

使用 select ‘payload’ into outfile ‘网站可访问的绝对路径’

1.知道绝对路径，且MySQL服务所属用户（默认为mysql）对根目录有可写权限。

2.数据库用户File权限 ，FILE权限指的是对服务器主机上文件的访问

3.GPC关闭，即能够使用单引号，方便写payload

4.MySQL账户有 File_priv 权限 判断： show variables like ‘%secure_file_priv%’;

  • 配置文件： –secure-file-priv

  • 该选项为空代表可以写入和导出任何目录文件

  • 该选项如果有文件夹目录则代表只允许写入和导出该目录下文件（PS：测试子目录也不行）

  • 该选项为null代表没有权限

  • 在 MySQL 5.5 之前 secure_file_priv 默认是空

    在 MySQL 5.5之后 secure_file_priv 默认是 NULL

UDF写入shell

使用 select ‘payload’ into outfile ‘插件目录的绝对路径’

1.知道绝对路径，且mysql服务所属用户需要有插件目录（ show variables like ‘%plugin%’ 查看插件目录绝对路径）的写入权限，linux上mysql的plugin目录默认为root所有，权限为755。

2.数据库用户File权限

3.MySQL账户有 File_priv 权限,需为空，即无限制导入导出

UDF的定义

简单的理解：UDF就是一个用户自定义的函数，拿来扩展数据库功能的

动态链接库

动态链接提供了一种方法，使进程可以调用不属于其可执行代码的函数。函数的可执行代码位于一个 DLL 文件中，该 DLL 包含一个或多个已被编译、链接并与使用它们的进程分开存储的函数。DLL 还有助于共享数据和资源。多个应用程序可同时访问内存中单个 DLL 副本的内容。
Windows 中，DLL 多数情况下是带有 “.dll” 扩展名的文件，但也可能是 “.ocx”或其他扩展名；Linux系统中常常是 “.so” 的文件。

提权原理

​ 而提权的操作为当我们拿到webshell时可以查看网页根部的配置文件，拿到Mysql中root的账号密码，利用root权限，创建带有调用cmd函数的’udf.dll’(动态链接库)。当我们把’udf.dll’导出指定文件夹引入Mysql时，其中的调用函数拿出来当作mysql的函数使用。这样我们自定义的函数才被当作本机函数执行。在使用CREAT FUNCITON调用dll中的函数后，mysql账号转化为system权限，从而来提权。

个人感觉跟suid提权差不多，都是借用一个高权限的功能来达到目的。

提权过程

查看secure_file_priv和plugin目录

要是遇到secure_file_priv不为空时，需将配置文件改为空，否则无法将动态链接库导入

如果遇见插件目录为空时，可以自己在MySQL的安装目录中生成\lib\plugin文件夹。寻找MySQL安装目录可用select @@basedir这条语句

这里提供大佬的动态链接库来用于导入。导入后使用CREATE FUNCTION sys_eval RETURNS STRING SONAME '链接库名';，即可像MySQL内置函数一样使用。

利用log写入

1.知道网站绝对路径

2.数据库用户对网站可访问路径的写权限

3.mysql连接用户有权限开启日志记录和更换日志路径/ROOT权限

报错注入中常用的函数

过滤or怎么查库

在 mysql => 5 的版本中存在库information_schema,记录着mysql中所有表的结构，通常，在mysql sqli中，我们会通过此库中的表去获取其他表的结构，即表名，列名等。但是这个库也会经常被WAF过滤。
在information_schema中，除了SCHEMATA，TABLES，COLUMNS有表信息外，高版本的mysql中，还有INNODB_TABLES及INNODB_COLUMNS中记录着表结构。

MySQL注入拿shell的过程

当存在SQL注入时，先查看当前用户的身份以及权限。

要是权限低的话可以尝试先读取数据库的结构、表、字段内容，要是运气好的话会发现有用户密码，再观察有无后台登陆之类的，也有可能获取shell。

要是权限高的话可以尝试有无文件读写的权利，命令执行、注册表读取之类的，也可以尝试使用日志提权，UDF提权等

MySQL注入绕WAF

屏蔽空格

使用/**/、制表符%09、换行符%0a、括号()绕过、反引号`绕过

等价函数字符替换绕过

用like或in代替=

逗号过滤：database(),1,1=database() from 1 for 1

等价函数

sleep函数可以用benchmark函数代替

ascii函数可以用hex、bin函数代替

group_concat函数可以用concat_ws函数代替

updatexml函数可以用extractvalue函数代替y

内联注释绕过

MySQL会执行放在/!…/中的语句。/!50010…/也可以，50010代表的是Mysql的版本号，当Mysql的实际版本大于内联注释中的版本，就会执行内联注释的代码

其他

大小写绕过、双写绕过、双重URL编码绕过，十六进制编码绕过、Unicode编码绕过(针对IIS中间件对Unicode自动编码)、ASCII编码绕过(SQL Server的char函数将ASCII码转换成字符)

文件上传

前端js验证绕过

js验证原理：应用程序在前端通过JS代码进行验证，而不是在后端靠逻辑来验证

绕过方式：屏蔽js或修改js代码

文件名过滤绕过

文件名过滤：通过文件后缀名来判断，在黑名单内的无法通过

绕过方式：修改文件后缀。eg:Php、phtml、php3、phps等

Content-Type过滤绕过

Content-Type用于定义网络文件的类型和网页编码，用于告诉服务器以什么方式读取上传的文件

不同的文件对应着不同的Content-Type，下面列出比较常用的

image/gif gif
image/jpeg jpeg、jpg、jpe

绕过方式：抓包修改Content-Type

文件头过滤

原理跟Content-Type过滤差不多，都是检测文件头的格式。

绕过方式：在文件原有的内容上添加文件头

常用的文件幻术头

474946383961 gif “GIF 89A”
474946383761 gif “GIF 87A”
FFD8FFFE00 .JPEG;.JPE;.JPG “JPGGraphic File”
504B0304 .zip “Zip Compressed”

可以留后门

把文件名包含1的解析成php
AddType application/x-httpd-php .png //png文件解析为php文件

php的一句话木马:
asp的一句话是:<%eval request (“pass”)%>
aspx的一句话是:<%@ Page Language=”Jscript”%> <%eval(Request.Item[“pass”],”unsafe”);%>

利用条件：将中间件的配置文件httpd.conf改为

AllowOverride All

并且去掉mod_rewrite.so的注释，开启rewrite模式

LoadModule rewrite_module modules/mod_rewrite.so

.user.ini文件

.user.ini的使用条件

  1. 服务器脚本语言为PHP
  2. 对应目录下面有可执行的php文件
  3. 服务器使用CGI／FastCGI模式

文件截断上传

当PHP版本低于5.3.4时，会将%00当成截断符，导致后面的数据直接被忽略，造成上传的文件被截断。要求上传文件的路径可控，不过这是个远古漏洞，没太大意义了

竞争条件文件上传

详解在条件竞争漏洞

文件包含

PHP中文件包含函数有以下四种

  • require()：只要程序一运行就包含文件，找不到被包含的文件时会产生致命错误，并停止脚本
  • require_once()：若文件中代码已被包含则不会再次包含
  • include():执行到include时才包含文件，找不到被包含文件时只会产生警告，脚本将继续执行
  • include_once()：若文件中代码已被包含则不会再次包含

漏洞分类

本地包含

读敏感文件，读PHP文件、包含日志文件GetShell、php://input等伪协议

若有phpinfo则可以包含临时文件，配合上传图片马，然后包含从而GetShell

无限制本地包含漏洞

这种就随便包含，无限制，可用伪协议什么的

有限制本地包含漏洞

在后端强行加上后缀名之类的

绕过方式

  • %00截断文件包含
    • 条件：magic_quotes_gpc = off、php版本<5.3.4
  • 路径长度截断文件包含
    • 原理：操作系统存在最长目录的限制，可以通过输入超过最长目录的路径长度，这样系统就会将后面的路径丢弃，导致后缀截断。windows下目录最大长度为256字节，linux下目录最大长度为4096字节
  • 点号截断文件包含
    • 原理：与路径长度截断文件包含相同

session文件包含

利用条件：当Session文件的内容可控，并且可以获取Session文件的路径，就可以通过包含Session文件进行攻击

获取Session文件的路径

  1. 通过phpinfo的信息可以获取
  2. 猜测默认Session存放位置 Linux：/var/lib/php/session

而Session文件的命名默认为sess_开头，后面接一串id值

日志文件包含

中间件日志文件包含：就如DC-5中，nginx含有两个默认日志：access.log、error.log，这些日志会将用户操作记录下来，只要写入恶意的操作代码，再包含日志文件即可达到目的

SSH日志包含：在远程连接时，将用户名改为恶意代码，无需考虑是否存在该用户，在ssh日志都会记录用户登录，ssh日志默认路径：/var/log/auth.log

远程包含

包含远程的木马或病毒文件，从而GetShell

要实现远程文件包含需要allow_url_fopen和allow_url_include都开启

无限制远程文件包含

与无限制本地文件包含相同

有限制远程文件包含

与有限制本地文件包含相同，绕过方式也相同

PHP伪协议

  • php://filter（本地磁盘文件进行读取）
    用法：?filename=php://filter/read=convert.base64-encode/resource=文件名

  • php://input

    可以访问请求的原始数据的只读流。即可以直接读取到POST上没有经过解析的原始数据。 enctype=”multipart/form-data” 的时候 php://input 是无效的。

    用法：?file=php://input 数据利用POST传过去。

    利用条件：需开启 allow_url_include，allow_url_fopen 不需要开启

    可写入恶意代码

  • data://
    利用条件：PHP版本>5.2.0

    如果传入的数据是PHP代码，就会执行此代码，可造成任意代码执行。

    data://text/plain;base64,dGhlIHVzZXIgaXMgYWRtaW4

  • file://伪协议

    file协议可以访问本地文件系统，读取到文件的内容。

    利用条件：不需要开启allow_url_fopen，不需要开启allow_url_include

命令执行

绕过空格

$IFS$9、%09、{}绕过：{cat,1.php}

绕过关键字过滤

  • 变量名拼接
    a=c;b=at;$a$b 1.php

  • 空变量绕过
    ca${x}t 1.php

  • \绕过
    c\at 1.php

  • 通配符绕过

    Linux支持通配符进行字符匹配，通配符的作用是用于模糊查询时通配文件名中某些不确定的字符。

    * 代表任意字符(0到多个)

    ? 代表一个字符

    [] 中间为字符组合，仅匹配其中任意一个字符

  • base64编码绕过

    `echo “aWQ=”|base64 -d`

XSS漏洞

原理

个人见解：当发现一个参数可控时，那么可以尝试输入恶意代码，这里的恶意代码大多数为JS

利用条件

要知道浏览器的内核版本

分类

反射型

经过服务器，不会存储在数据库

非持久型xss攻击是一次性的，仅对当次的页面访问产生影响。非持久型xss攻击要求用户访问一个被攻击者篡改后的链接，用户访问该链接时，被植入的攻击脚本被用户游览器执行，从而达到攻击目的

存储型

存储XSS，会把攻击者的数据存储在服务器端，攻击行为将伴随着攻击数据一直存在
通常存在场景：博客评论区、能储存到数据库且其他用户也能访问的页面

DOM型

SSRF(未完成)

SSRF漏洞形成的原因是因为应用程序存在可以从其他服务器获取数据的功能，但是服务器的地址并没有做严格的过滤，导致应用程序可以访问任意的URL链接，攻击者通过精心构造URL链接，可以通过SSRF漏洞进行以下攻击：

  • 可以通过服务器，通过服务器获取内网主机、端口和banner信息。
  • 对内网的应用程序进行攻击，比如Redis、jboss等。
  • 利用file协议读取文件。
  • 可以攻击内网程序造成溢出。

常见的有gopher+ssrf来写shell或者redis未授权访问
