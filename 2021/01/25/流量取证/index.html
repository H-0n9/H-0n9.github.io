Misc中的流量取证

发表于2021-01-25|更新于2021-01-25
|字数总计:596|阅读时长:2分钟|阅读量:

流量分析作用就是溯源攻击流量，找到攻击者的IP

Wireshark的使用方法

想要查看响应的数据包可以在相应的地方右键使用导出分组字节流

过滤器分为显示过滤器和捕获过滤器
ip: ip.src == 127.0.0.1 ip.dst == 127.0.0.1
mac: eth.dst == mac地址 eth.addr == mac地址
port: tcp.srcport == 80 tcp.dstport == 80 tcp.port == 80
长度: udp.length == 20 筛选长度为20的udp流量包
http: http.request.method == “GET” 筛选http的请求方式
http.request.url == “/img” 筛选想用数据包中含有相应路径
http.contains “flag” 筛选相应内容中含有相应的文本
还可以使用逻辑运算符and、or

tshark是wireshark的命令行方式，相关命令参数可在网上查询

协议分析

HTTP协议
一般查看流量包做了操作，上传了什么文件，保存在本地后查看

HTTPS协议
服务端和客户端的信息传输都会通过TLS进行加密，所以传输的数据都是加密后的数据，所以得拿到密钥后导入wireshark可以看到明文的内容，但是这密钥是用公钥加密的，拿不到密钥，需要题目提供或通过其他手段才能拿到密钥

USB协议
数据一般在leftover Capture Date中，每一个数据包的数据区有四个字节(可能是8个字节)，第一字节代表按键，当取0x00时，代表没有按键、为0x01时，代表左键，0x02时，代表右键。第二个字节可以看成是一个signed byte 类型，其最高位为符号位，值为正时，代表鼠标水平右移多少像素，为负时，代表左移多少像素。第三个字节与第二字节相似，代表上下移动
用tshark将鼠标数据提取出
tshark -r 流量包名 -T fields -e usb.capdata > micedata

键盘协议同理，可以使用脚本获得https://github.com/WangYihang/UsbKeyboardDataHacker

WiFi协议
要抓取到握手包才能通过爆破获得WiFi密码

  • BSSID：路由器、AP的MAC地址
  • PWR：信号强度，越小越强
  • Data：传输的数据大小
  • CH：无线信道，要看准
  • ENC：加密协议
  • ESSID：WiFi名称

对流量包爆破密码
aircrack -ng xxx.pcap -w /爆破字典的路径

用密码解密流量包
airdecap -ng [capfile] -e [ESSID] -p [password]
